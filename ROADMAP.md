# MOI Benchmark Roadmap

MOI Benchmark 由 Astra、Memoria、文档解析、RAG、NLP2SQL 五个独立 Track 组成。每个 Track 独立完成方案审阅、资产建设、试运行、报告发布和持续回归；仓库治理与公共基础设施由项目统一维护。

> Astra 与 NLP2SQL 的现有方案尚未审阅。本路线图只记录其推进阶段，不采纳草稿中的竞品、任务数量、指标、排期或资源细节。

## 项目级里程碑

### M0：仓库基线（当前）

- [x] 建立可长期维护的目录和证据链
- [x] 将五个功能 Track 纳入统一路线图
- [ ] 完成 GitHub 仓库与项目治理配置

### M1：Track 方案基线

- [ ] 五个 Track 均有负责人
- [ ] 五个 Track 的方案均完成审阅并形成批准版本
- [ ] 公共元数据、数据治理和发布规则完成审阅

### M2：Track 试运行

- [ ] 五个 Track 分别完成版本冻结与试运行
- [ ] 每个试运行均生成可追溯的 `run_id` 和复核记录
- [ ] 试运行暴露的问题均进入 GitHub Issues

### M3：首个完整版本

- [ ] 五个 Track 分别发布审阅通过的 Track 报告
- [ ] 发布 MOI Benchmark 综合报告
- [ ] 通过 tag 和 GitHub Release 冻结方案、版本、运行记录与报告

### M4：持续回归

- [ ] 各 Track 建立分级回归集和执行频率
- [ ] 自动生成趋势、退化告警和结果差异摘要
- [ ] Track 方案、参评对象、数据集和许可证定期复核
- [ ] 任务与评分器变更产生新版本，不改写历史结果
